Process Doc
01
The Talent Pool
Candidates. Skills. Availability.

A custom Candidate__c object manages the full freelancer roster — 11 fields capturing skills, sprint rates, availability status, and MENA location data.

  • Multi-select Skills picklist — 8 categories including LLMs, MLOps & UI/UX
  • Availability stages — Available, In Sprint, Placed, Inactive
  • Sprint Rate (USD) and LinkedIn Profile fields
  • MENA Based checkbox for regional filtering
02
The Matcher
Brief. Match. Deploy.

A guided Talent Matcher Screen Flow walks the recruitment team through the full client briefing process — from selecting a client and region to choosing an UpPaq service package.

  • Screen 1 — Client Brief: client, region & skills required
  • Screen 2 — UpPaq Selection: service package & sprint duration
  • Screen 3 — Confirmation summary before submission
  • Auto-creates Engagement record with Status set to Briefing
03
The Pipeline
5 Engagements. Full Lifecycle.

The Engagement__c object links every Candidate, Client, and UpPaq into a single placement record — auto-numbered ENG-0001 through ENG-0005 — tracked across a 5-stage pipeline.

  • Stages: Briefing → Matching → Active Sprint → Completed → Cancelled
  • Lookup relationships to Candidate, Account & UpPaq
  • Region picklist — UAE, KSA, Egypt, Qatar, Kuwait, Other MENA
  • Completed status triggers automated follow-up task
04
The Dashboard
MENA Pipeline. At a Glance.

An executive dashboard embedded as the app home page surfaces placement performance, revenue by client, and candidate availability — built for C-suite visibility across the MENA region.

  • Active Sprints — total pipeline value metric ($31k)
  • Revenue by Client — bar chart across all 5 MENA accounts
  • Active Engagements by Region — UAE vs KSA donut chart
  • Candidate Availability — Available, In Sprint, Placed breakdown
